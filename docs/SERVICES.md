## md-core

**Repo**: https://github.com/FagorHealthcare/md-core
**Image**: `gailen/md-core` (DockerHub)
**Container port**: 8080
**Replicas**: 2 (dev-0, pre); rolling update strategy

### Stack

- Java 17 with `--enable-preview`
- Quarkus 2.9.2.Final
- Maven (`./mvnw`)
- PostgreSQL via Hibernate ORM with Panache (active record)
- Flyway 38 versioned migrations (`src/main/resources/db/migration/`); `migrate-at-start=true`
- SmallRye JWT (RS256, key files mounted from `ssh-key-secret`)
- SmallRye Reactive Messaging — MQTT (Mosquitto via Eclipse Paho)
- Quartz, clustered, JDBC store (lives in same Postgres)
- Twilio Java SDK 9.5.0
- Quarkus REST Client Reactive
- Apache POI, Apache Commons CSV, libphonenumber, Lombok
- Tests: JUnit 5 + Karate 1.2.1.RC2 + REST-Assured + Mockito; JaCoCo coverage

### Top-level packages (`src/main/java/com/medicaldispenser/`)

| Package | Purpose |
|---|---|
| `conversation/` | WhatsApp message routing, content templates, conversation state |
| `jobs/` | Quartz triggers for reminders, missed-dose checks, hourly sweeps |
| `log/` | Activity log endpoints (`/log/activity`, `/log/data`) |
| `onboarding/` | Patient tracking setup (`/seguimiento`, `/pauta`) |
| `online/` | Pharmacy auth context, custom collaborators (consumes `auth-library`) |
| `repo/` | Panache entities + repositories: `Tratamiento`, `Blister`, `Toma`, `Seguimiento`, `Activacion`, `MessageLog`, `ActivityLogEntry`, `PhoneConfiguration`, `FMDStatsEntry`, `BlisterToken`, `Dummy` |
| `services/` | Business logic: `TwilioNotificationService`, adherence calculation, user mgmt |
| `shc/` | SHC device integration: MQTT emitter (`ShcConfigurationEventsEmitter`), command/event endpoints |
| `sync/` | Pharmacy sync API implementation (`SyncApi` from `sync-api-spec`) |
| `user/` | User removal, treatment deletion, tracking config |

### REST controllers (JAX-RS)

| Path prefix | Controller | Notes |
|---|---|---|
| `/wu/*` | `WhatsappController` | Twilio webhook receiver |
| `/test/*` | `TestController` | Debug/diagnostic endpoints (disabled in prod via profile) |
| `/seguimiento/*` | `OnboardingController` | Patient tracking lifecycle |
| `/sync/*` | `SyncController` | Implements `SyncApi` from `sync-api-spec/index.yaml` |
| `/shc/*` | `ShcController` | Device events & commands |
| `/user/*` | `UserController` | User removal, tracking config |
| `/log/*` | `LogController` | Activity logging |
| `/q/*` | Quarkus built-ins | `/q/health`, `/q/health/live`, `/q/health/ready`, `/q/metrics`, `/q/openapi`, `/q/swagger-ui` (dev only) |

### Build & run

```bash
# JVM build (default — fast-jar layout)
./mvnw package

# Build + push container via JIB (CI uses this)
./mvnw package \
    -Dquarkus.container-image.build=true \
    -Dquarkus.container-image.push=true \
    -Dquarkus.application.version=main.247

# Native (rarely used)
./mvnw package -Pnative

# Dev mode (hot reload, listens on :8080)
./mvnw quarkus:dev

# Run a single test
./mvnw test -Dtest=FMDEndpointTest
```

### Configuration

- `src/main/resources/application.properties` (with `%dev.`, `%staging.`, `%prod.` profile prefixes)
- Runtime overrides come from a Kubernetes ConfigMap (`md-core-config`) generated by Kustomize from `k8s/environments/<env>/application.properties` and mounted as `/work/config/application.properties`
- Key knobs: WhatsApp Twilio creds, MQTT broker URL, JWT issuer, Postgres URL, swagger toggle, ngrok (dev only)

### Dockerfile

`src/main/docker/Dockerfile.jvm` (active) — base `registry.access.redhat.com/ubi8/ubi-minimal:8.4`, Java 15 OpenJDK headless, runs as UID 1001, exposes 8080. Entrypoint `/deployments/run-java.sh`.

JIB plugin (Quarkus-managed) builds with base `eclipse-temurin:17.0.5_8-jdk-ubi9-minimal`.

### CI/CD

- **`.github/workflows/cd.yaml`** (push to any branch) — build, push image as `main.<run>`; on `main` branch additionally `kubectl set image` against `md-dev-cluster` and re-tag as `dev`.
- **`.github/workflows/release.yml`** (manual) — deploys an existing tag to `md-pre-cluster` (production), re-tag as `prod`.
- **`.github/workflows/dev-release.yaml`** (manual) — same as `cd.yaml` deploy job but for explicit re-deployment of an arbitrary tag to dev.
- **`Jenkinsfile`** — legacy path that targets AWS EKS `eu-west-1`. Still wired but the active path is GitHub Actions to DOKS.

## md-auth

**Repo**: https://github.com/FagorHealthcare/md-auth
**Image**: `gailen/md-auth`
**Maven artifact**: also publishes the `auth-library` JAR to GitHub Packages (consumed by `md-core` and `md-resi-back`)

### Stack

- Java 17, Quarkus 2.16.12.Final, Maven multi-module
- Modules: `auth-library/` (shared API + DTOs + JWT helpers) and `auth-back/` (the deployable service)
- Hibernate Panache, Flyway (`flyway_schema_history`), separate from md-core's history
- SmallRye JWT
- Apache POI (Excel for document acceptance)

### Surface

REST controllers under `auth-back/`:

- **`AuthController`**: `/auth`, `/autologin`, `/checkauth`, `/external-user/{provider}/{externalId}`, `/profile`, `/users` (CRUD), `/request-password-reset`, `/reset-password`, `/sign`, `/signed-documents`, `/unlock`, `/docs-pending/{userId}`, `/download-doc/{docId}`, `/exist-by-login/{login}`, `/add-user-to-category`, `/renew`
- **`FmdAuthController`**: `/fmd/activation`, `/fmd/refresh`, `/fmd/auth`, `/fmd/check-token`, `/cinfa/activate/{cinfaCode}`, `/cinfa/activateByFmd/{fmdId}`

Roles: `ROLE_PHARMACY`, `ROLE_PHARMACY_REFRESH`, `ROLE_USERMANAGEMENT`, `ROLE_USERMANAGEMENT_REFRESH`.

### Cinfa Salesforce integration

- OAuth 2.0 client_credentials grant against `https://cinfa.lightning.force.com/services/oauth2/token`
- Sandbox: `https://cinfa--dev.sandbox.my.salesforce.com/services`
- Configured via `application.properties`: `client_id`, `client_secret`, `username`, `password`
- Used in `CinfaCommunityService` to mark pharmacy activations complete

### Build & run

```bash
./mvnw package
./mvnw quarkus:dev    # auth-back/ subdir
java -jar target/quarkus-app/quarkus-run.jar
```

### CI/CD

- `.github/workflows/cd.yaml` — build, push image, on `main` deploy to `md-dev-cluster`. Also publishes `auth-library` to GitHub Packages.
- `.github/workflows/release.yml` — manual production deployment (to `md-pre-cluster`).

## md-resi-back

**Repo**: https://github.com/FagorHealthcare/md-resi-back
**Image**: `gailen/md-resi-back`

### Stack

- Java 17, Quarkus 2.16.12.Final
- OpenAPI-driven: generates JAX-RS interfaces from `md-resi-api-spec/index.yaml` via `openapi-generator-maven-plugin` — implementations live in `com.medicaldispenser.resi.front.http`
- Consumes `auth-library` from md-auth
- ANTLR 4.11.1 (grammar parsing — likely prescription syntax)
- Apache POI for Excel exports

### External integrations

- **AEMPS** — Spanish drug regulator's CIMA REST API (`https://cima.aemps.es/cima/rest`) for medication lookup by national code (CN). Wrapped by `AEMPSAPI` REST client and exposed via `AEMPSController`.
- **Cinfa Salesforce** — same credentials as md-auth (passes-through for some flows).

### Database

Same Postgres cluster as md-core/md-auth, but separate Flyway history table: `flyway_resi_schema_history`.

### Build & run

Same Quarkus pattern. The only special step is API generation, which is automatic during `mvn package`.

## md-pwa

**Repo**: https://github.com/FagorHealthcare/md-onboarding-pwa
**Image**: `gailen/md-pwa` (containerized) + Firebase Hosting (legacy Jenkins path)

### Stack

- Angular 12.0.1 + Ionic 5.5.2 + Capacitor 3.0.2
- TypeScript 4.2.4
- Angular Service Worker (NGSW) — production only
- @ngx-translate (5 languages: es, en, pt, it, fr)
- Angular Material 12, lodash-es, ngx-avatar
- API client auto-generated from `client-api-spec-circupack/index.yaml` (git submodule) via `openapi-generator-cli`
- Tests: Karma + Jasmine + Protractor (e2e)

### Routes

```
/           → redirects to /blister
/blister    → BlisterPageModule (active)
/install    → InstallPageModule (PWA install prompt)
/home       → HomePageModule (commented out)
```

### Runtime config injection

`src/config/config.js` is loaded before Angular bootstrap and assigns `window.EXTERNAL_CONFIG = { API_URL, ANALYTICS_ID }`. Falls back to `environment.ts`. In Kubernetes this file is mounted from a ConfigMap (`md-pwa-config`), validated by an init container running `node:alpine` that does a JS syntax check before the nginx pod starts.

### Build & run

```bash
npm install
npm run build-api      # Regenerate API client from OpenAPI spec
npm start              # Ionic dev server
npm run build          # Production build to www/
```

### Container

`Dockerfile` — `nginx:1.21.3-alpine`, listens on 8080, SPA routing via `try_files $uri /index.html`, gzip enabled.

### CI/CD

- `Jenkinsfile` — Firebase Hosting deploy (legacy; conditionally `ionic build --prod` for branches matching `demo/pro/`)
- `.github/workflows/cd.yaml` — Docker build + push, deploy to `md-dev-cluster` on `main`

## md-backup

**Repo**: https://github.com/FagorHealthcare/md-backup
**Image**: `gailen/md-backup`
**Runs as**: Kubernetes CronJob

### Stack

- Alpine Linux container
- Bash scripts
- Tools installed: `kubectl`, `kubectx`, `postgresql-client`, `rdiff-backup`, `s3fs`
- Multi-stage Dockerfile (builder → s3installer → final)

### What it backs up

- PostgreSQL dump of the env's database
- Kubernetes resource descriptors (`kubectl get -A -o yaml`)
- Destination: S3 bucket per env (`medicaldispenser-dev-backup`, `medicaldispenser-backup`)

### Schedule

- dev-0: `0 4 * * *` UTC
- pre: `0 3 * * *` UTC

### Permissions

CronJob's ServiceAccount is bound to `cluster-admin` ClusterRoleBinding (so it can read all resources). Container runs privileged (FUSE for s3fs).

### CI/CD

None — image must be built and pushed manually.

## API spec repos

### sync-api-spec

OpenAPI 3.0.2 contract between pharmacy backends and `md-core`. Defines:

- `/sync/ping`, `/sync/carga`, `/sync/auth`, `/sync/check`, `/sync/shc`
- `/shc/event`, `/shc/command`
- `/user/invite`, `/user/paciente`, `/user/seguimiento`
- `/log/data`, `/log/activity`

Plus a separate `fmd-auth-index.yaml` for FMD auth endpoints. Consumed as a git submodule and via `openapi-generator-maven-plugin` in md-core.

### md-resi-api-spec

OpenAPI for the residence backend. Defines patient/medication/prescription/doctor/document endpoints. Consumed by md-resi-back (server interfaces) and md-resi-front (client).
